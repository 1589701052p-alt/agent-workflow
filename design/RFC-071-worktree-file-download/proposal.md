# RFC-071 — 工作目录文件下载（任务详情页）

## 背景

RFC-065 在任务详情页加了「工作目录」页签：左侧懒加载目录树，右侧把选中文件按纯文本渲染进 `<pre>`。
为防止把巨型文件灌进浏览器，预览有 2 MiB 上限（`WORKTREE_FILE_MAX_BYTES`）——超过的文件后端返回
`oversized: true`、`content: ''`，前端只展示一行「文件过大，未预览」的提示。

问题：**用户无法把文件内容从 UI 里取出来**。

- 小文件：能看见，但要复制粘贴整段才能落盘，不可靠（`<pre>` 选区受限、二进制乱码）。
- 大文件（构建产物、数据 fixture、日志、二进制）：连看都看不到，更别说拿出来——只能去磁盘上手动定位
  `~/.agent-workflow/worktrees/{repo-slug}/{task-id}/...`，对远程 daemon 场景几乎不可行。

## 目标

- 在右侧「打开的文件」预览区**右上角**加一个下载按钮，下载当前打开的那个文件。
- 普通（可预览）文件能下载。
- **超过预览上限、无法显示的文件也能下载**（这是本 RFC 的核心诉求）。
- 下载的是文件**原始、完整、逐字节**的内容，文件名取其 basename。

## 非目标

- 多文件 / 文件夹 / 打包（zip）下载。
- 从目录树某一行直接下载（只下载右侧已打开的那一个文件）。
- 断点续传、Range 请求、下载进度条（按钮 loading/disabled 态足够）。
- 后端新增接口——复用 RFC-005 已有的原始字节流式端点。

## 用户故事

1. 我在工作目录页签点开一个 `config.yaml`，预览正常显示；我点右上角的 ↓，浏览器把
   `config.yaml` 存进下载目录。
2. 我点开一个 50 MiB 的构建产物，右侧显示「文件过大，未预览」；我仍能点 ↓ 把完整文件下载下来。
3. 我把 daemon base URL 指向了一台远程机器（远程 daemon 模式）；下载依然能用，且 token 不出现在 URL 里。

## 验收标准

- **AC1**：普通预览态与「过大」态右上角都出现下载按钮。
- **AC2**：点击后浏览器以文件 basename 为文件名保存该文件。
- **AC3**：过大文件下载到的是**完整字节**（不被截到 2 MiB），与磁盘上原文件一致。
- **AC4**：同源（本地 daemon）与跨源（远程 daemon base URL override）都可用；鉴权走
  `Authorization` 头，token 不进 URL。
- **AC5**：下载失败（401 / 404 / 网络）时给出**就地错误提示**，不静默失败。
- **AC6**：下载进行中按钮置为进行态并禁用，避免重复触发。
